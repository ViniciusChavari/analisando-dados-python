# Análise de Dados: Cancelamento de Clientes (Churn)


> Projeto desenvolvido durante Jornada Python para identificar os principais motivos de cancelamento em uma base de dados de clientes e propor soluções estratégicas.


## Cenário e Objetivo
O objetivo principal foi analisar uma base de dados com informações de clientes para entender por que alguns estavam cancelando o serviço. 
Através desta análise, foi possível identificar padrões e sugerir ações para reduzir o *churn* (taxa de cancelamento).

## Tecnologias e Bibliotecas
- **Python**: Linguagem principal.
- **Pandas**: Utilizada para a manipulação e tratamento da base de dados.
- **Plotly**: Utilizada para a criação de gráficos interativos e visualização de dados.
- **Jupyter Notebook/Google Colab**: Ambiente de desenvolvimento.

## Metodologia Aplicada
Seguindo as boas práticas de **Engenharia de Dados**, o projeto foi dividido em:
1. **Importação e Visualização:** Carga dos dados e entendimento das variáveis disponíveis.
2. **Data Cleaning:** Remoção de informações inúteis e correção de valores vazios.
3. **Análise Inicial:** Cálculo da percentagem de cancelamentos para entender o tamanho do problema.
4. **Análise Profunda:** Cruzamento de informações para descobrir a "causa raiz" dos cancelamentos.

## Insights e Resultados
Através dos gráficos gerados, foi possível observar que:
- Clientes com contrato mensal têm uma taxa de cancelamento muito maior.
- Problemas no setor de atendimento impactam diretamente na retenção.
